# AGENT.md

Короткая памятка по проекту `analytics-methods`.

## Назначение

Node.js приложение с HTTP API и фронтендом для демонстрации аналитических методов и визуализаций.

## Основные файлы

- `src/server.js` - сервер и API
- `src/analysis.js` - сборка аналитики
- `src/data.js` - источники данных, конфиг, подключение к БД
- `src/migrate.js` - раннер миграций
- `migrations/*.js` - миграции
- `config/access.yml` - реальный конфиг
- `config/access.yml.dist` - пример конфига

## Источники данных

- `excel` с demo fallback
- `yahoo`
- `market-api`

Примечание: Yahoo может отвечать `429 Too Many Requests`.

## База данных

Использовать только текущее подключение приложения из `config/access.yml`.
Не хардкодить доступы.

Проверка БД:
- функция: `checkDatabaseConnection()`
- API: `/api/db-check`

## Миграции

Всегда использовать текущее подключение приложения.

Команда:

```bash
npm run migrate
```

Служебная таблица миграций:
- `migrations`

## Правила проекта

- любые изменения схемы БД делать только миграциями
- реальные секреты хранить только в `config/access.yml` или env
- `config/access.yml` не коммитить
- новые конфиги и интеграции сначала отражать в `config/access.yml.dist`
- если внешний API не подходит по типу данных, не подменять им рыночный источник молча
- если Yahoo не отвечает, прямо писать, что это лимит источника
- если просят добавить настройку и явно не указан другой способ хранения, по умолчанию добавлять её в таблицу `settings`
- при создании любой новой таблицы последние две колонки всегда: `created_at`, `updated_at`
- время от времени коммитить изменения в репозиторий и пушить их

## Таблица settings

Текущая структура:
- `id`
- `name`
- `val`
- `created_at`
- `updated_at`

## Перед завершением работы

Проверять:

```bash
npm test
npm run migrate
```
